## 저장 구조

현재 데이터는:
- 브라우저 localStorage 기반 저장
- 입력 즉시 자동 저장
- 별도 저장 버튼 없음

동작 흐름:
입력 변경
→ state 변경
→ useEffect 감지
→ localStorage 자동 저장

# 핵심 운영 철학

이 시스템은 단순 일정관리 앱이 아니다.

목표는:
"출강 운영 인텔리전스 시스템"

즉:
- 일정
- 재고
- 이동
- 비용
- 정산
- 관계관리
- 운영효율

을 하나로 연결하는 구조를 목표로 한다.

---

# 중요한 아이디어 및 향후 핵심 기능

## 주소 자동 검색 API
기관명을 입력하면:
- 주소 자동 입력
- 지도 검색 자동화

예:
전주대정초등학교
→ 전북특별자치도 전주시 완산구 맏내로 57

예상 구현:
- 카카오맵 API
- 네이버 지도 API

---

# 지도 기반 운영 기능

## 길찾기 기능
상세페이지 주소 옆:
- 네이버 길찾기 버튼
- 카카오 길찾기 버튼

클릭 시:
- 출발지~도착지 자동 연결
- 바로 길안내 가능

---

## 이동 분석 기능
주소 기반으로:
- 거리 계산
- 이동시간 계산
- 예상 출발시간 계산
- 유류비 계산
- 출장비 계산

---

## 날씨 연동
출강 지역 기준:
- 비예보
- 폭설
- 기상특보

표시 가능

---

## 지역 분석
출강 데이터 누적 시:
- 지역별 출강 횟수
- 매출 지역 분석
- 이동 효율 분석

가능

# 중요한 개발 방향

현재는:
"기능 안정화 + 운영 구조 설계"

단계이다.

아직 서버/로그인/클라우드보다:
- 실제 운영 흐름
- 업무 구조
- 데이터 구조

를 먼저 안정화하는 것이 중요하다.

# 현재 시스템의 성격

현재 시스템은:
"로컬 기반 출강 운영 시스템 MVP"

단계이다.

하지만 방향성은:
- 단순 메모앱

실제 현장 운영 최적화 시스템을 목표로 한다.
